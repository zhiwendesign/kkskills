# Diagram Board Design Rules

## Core Mood

The visual language should feel:

- structured
- visual
- teachable
- modular
- lively but controlled

It should not feel:

- like a plain document
- like a comic cover
- like a minimal keynote
- like a scrapbook collage

## Palette

Use this order of importance:

1. `#f4fbff`, `#ffffff`, `#dff3ff`, `#e7f8e9`, `#fff2bf` as main colors
2. `#314253` as softened outline and text
3. `#44a8ff`, `#78c97b`, `#ffcc4d`, `#ff8b7f` as tag and marker colors

## Layout Bias

The default `3:4` poster should feel like a clean explainer board:

- one large title block
- one upper-right metric card
- one highlighted summary strip
- one left panel with clear step cards
- one right panel with node cards or supporting modules
- one lifted footer note area

## Diagram Behavior

- use connectors, arrow hints, dotted guides, or numbered nodes
- modules should feel related, not randomly stacked
- let the reading order be obvious from top-left to lower-right
- avoid overly dense text inside each node

## Date Sticker

- keep it near the upper-right
- let it sit across the board edge like a pinned label
- use `YYYY年MM月DD日`

## Contact Cluster

- use `assets/contact-code.png` and `assets/kk-character.jpeg`
- keep both cards the same size
- place the QR to the left of the IP
- keep `合作交流` and `是卡卡呀` as compact captions below

## Final Check

- the explainer-board feeling is obvious
- the title and metric card establish hierarchy quickly
- modules feel connected by logic or visual hints
- the date sticker is visible and correctly formatted
- the QR and Kaka IP are both present
- the two contact cards are the same size
