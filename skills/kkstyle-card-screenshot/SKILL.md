# KKStyle Card Screenshot

## Overview

这个 skill 用来把原始信息做成“可爱、厚边、亮蓝底、海豹探头”的卡片，而不是社论风、企业风或极简海报。
默认优先产出更饱满的 `3:4` poster 型结构：大标题、右上角突破白卡的重点贴纸、中段双栏信息块、上提的底部信息条，以及固定的二维码 + IP 右下角组合。

默认保持三个交付阶段，除非用户明确要求跳过：
1. 先用一句话判断信息密度。
2. 输出完整 HTML，包含内嵌 CSS。
3. 明确自检文字在移动端缩放后仍然清晰，装饰不会压住信息。

## Workflow

### 1. 先判断卡片类型

先看内容适合哪种 kkstyle 卡片：

- 推荐海报型：适合工具、课程、资源推荐，默认使用。
- 摘要海报型：一个主观点 + 两栏支持信息。
- 清单海报型：一个强标题 + 一个数字贴纸 + 多个短句块。
- 说明海报型：顶部强标题，下方双栏或三块信息区。

信息少时，也不要只剩大片留白。优先补摘要气泡、小卡片、结论条和底部说明块。
信息多时，优先增加块级层次，不要塞满一堆细字。

### 2. 先定比例，再定阅读路径

kkstyle 不适合把一个通用网页硬缩放成所有比例。要先根据比例重排：

- `3:4`：默认主结构。顶部海豹造型 + 稍微下沉的大标题 + 右上角爆点贴纸 + 摘要气泡 + 中段双栏 + 上提的底部信息条 + 固定二维码和 IP。
- `4:3`：适合横版摘要，卡面可以偏左，右侧加贴纸信息列。
- `1:1`：适合社媒方卡，标题要更集中，贴纸块不要超过四个。
- `9:16`：适合竖版封面，标题分成两段，底部留一块醒目的行动标签。
- `16:9`：适合视频封面或横幅，减少段落长度，更多使用大词组和短句。

精确尺寸见 [references/layout-presets.md](references/layout-presets.md)。

### 3. 遵循 kkstyle 视觉规则

默认视觉语言：

- 主背景用亮蓝 `#276BEA`。
- 外围带像素格或方块感边框，颜色偏浅蓝 `#A4CEFF` / `#62AAFF`。
- 主卡面用偏白 `#FCFCFC`，带厚黑边 `#1B1B1B`。
- 卡片顶部做出“海豹探头”轮廓：两个圆弧耳朵或鼓包，加两道眉形装饰。
- 在眼睛最右侧固定放一个萌感日期贴纸，格式用 `YYYY年MM月DD日`，并让它半挂在白色卡片上沿。
- 至少有一个重点模块要像贴纸一样抢眼，优先放右上角，并允许局部突破白卡边界。
- 点缀色只少量使用：橙 `#FF4D00`、黄 `#FFED00`、荧光绿 `#C2FF2E`。
- 整体气质要可爱、亲切、贴纸化，不要企业感，也不要赛博霓虹感。

详细规则见 [references/kkstyle-design-rules.md](references/kkstyle-design-rules.md)。

此外，每张海报都必须带上固定右下角组合：

- 使用打包资产 `assets/contact-code.png` 作为固定二维码位。
- 默认放在 IP 左侧，下方文案固定为 `合作交流`。
- 二维码卡和 IP 头像卡保持同尺寸。
- 二维码、IP、caption 都必须保持在最上层，不能被白色主卡边界裁掉。
- 使用打包资产 `assets/kk-character.jpeg`。
- 默认放在画面右下角，做成角色贴纸、角色框或头像卡。
- 默认排成上下结构：上面角色图，下面文案 `是卡卡呀`。
- 角色 caption 应该简洁、克制，优先用小条形标签，不要做得比主信息还花。
- 角色位不能省略，除非用户明确要求隐藏 IP。

日期贴纸也默认固定显示：

- 放在顶部眼睛区域最右侧。
- 位置要往下落在白色卡片边界上，不要整块悬空。
- 优先使用内容里的明确发布日期、更新时间或事件日期。
- 如果内容没有明确日期，再使用当天日期。
- 日期只用 `YYYY年MM月DD日` 这一种格式，不要混用短横线或斜杠。

### 4. 写截图优先的 HTML

输出给截图器的 HTML 必须是固定画布：

- 使用精确像素尺寸，不依赖 `100vw` / `100vh` 作为主布局。
- 清空默认边距：`html, body { margin: 0; }`。
- 背景层、像素边框层、主卡层分开。
- 主卡层优先保证标题可读，再摆贴纸和标签。
- 装饰元素只能辅助气氛，不能挡住正文。
- 固定预留右下角 IP 区域，避免正文挤占角色位。
- 对 `3:4` 海报，优先用“标题区 + breakout 重点卡 + 摘要气泡 + 双栏主内容 + 底部收尾”的顺序排版。
- 标题要给海豹顶部和标签留呼吸感，不要紧贴上缘。
- 顶部日期贴纸要落在白色卡片上沿附近，不能压到眉毛或标题。
- 左下角的 `项目地址` / `一句话建议` 一类信息条要适当上提，避免底部发空。

kkstyle 推荐这些结构元素：

- 厚描边标题
- 贴纸标签
- 漫画气泡或注释条
- 右上角 breakout 数字卡或重点卡
- 双栏信息块和底部特性条
- 小方块像素阵列
- 轻微旋转的模块，但不要转太多

### 5. 内容布局规则

特别注意：

- 标题要短、狠、可爱，优先做 1 到 3 行。
- 正文不要太长，单段最好控制在 30 到 60 个汉字量级。
- 次级信息更适合做列表块、标签块、短句块。
- 当内容太多时，先归纳成三到五个点，再排版。
- 卡片底部可以放“提醒贴纸”或“结论条”，但不要抢主标题。
- `是卡卡呀` 应该作为角色介绍贴纸，而不是主标题的一部分。
- 版面看起来太空时，优先补支持模块，不要只把标题字号做大。
- 右上角重点贴纸要承担“先抓眼球”的作用，不要缩回成普通信息块。
- 底部二维码和 IP 的标签都要轻一点，作用是说明，不是第二主标题。

### 6. 需要 PNG 时再截图

如果用户要求图片输出，先保存 HTML，再运行：

```bash
./scripts/capture_card.sh input.html output.png 3:4
```

脚本会自动查找 Chrome / Chromium，并在 `--headless=new` 失败时回退到 `--headless`。
如果最终 HTML 不在 `assets/` 目录下，优先把 `assets/kk-character.jpeg` 内联成 data URI，可用：

```bash
./scripts/image_to_data_uri.py assets/kk-character.jpeg
```

## Output Contract

处理 kkstyle 卡片请求时：

1. 第一行只写一句话，判断信息密度。
2. 然后输出完整 HTML 代码块。
3. 如果用户还要图片，HTML 后补一行说明比例和截图命令。

解释尽量短，HTML 才是主要交付物。

## Failure Checks

出现这些情况时，结果不合格：

- 蓝底没有形成明显风格，最后看起来像普通海报。
- 海豹造型缺失，或者只是一个普通圆角矩形。
- 黑色描边太弱，导致画面没有贴纸感。
- 文字被贴纸、气泡、像素块挡住。
- 信息太多却没有做摘要，结果一整页都在挤小字。
- 版面大面积发空，看起来像没排满的半成品。
- 右上角重点模块不够醒目，或者完全缩回白卡内部。
- 标题过于贴近顶部装饰，显得发挤。
- 顶部日期贴纸缺失、格式不对，或者压住了眼睛和眉毛区域。
- 底部信息条太靠下，和右下角角色位脱节。
- 二维码或 IP 被白色主卡边界压住、裁切，或者两者尺寸明显不一致。
- 配色用了太多杂色，丢掉了蓝、白、黑为主的统一感。
